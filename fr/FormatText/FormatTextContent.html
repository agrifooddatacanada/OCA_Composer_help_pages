Règles de format


Lorsque vous disposez d'un ensemble de données, vous pouvez avoir des attentes quant au type de données à saisir pour chaque attribut.

Les règles de définition des données acceptables sont appelées règles de format. Dans le moteur sémantique, les utilisateurs peuvent sélectionner des règles dans une liste pour chacun de leurs types de données.

Booléen

Les données booléennes sont un type de données qui représente deux valeurs possibles : vrai ou faux.

DateTime

Sélectionnez une règle de format DateTime dans le menu déroulant. Il est préférable d'utiliser une norme ISO pour votre représentation DateTime.

En arrière-plan, les règles de format pour DateTime sont exprimées dans le langage RegEx (expression régulière), en particulier la variante Rust RegEx. Il n'est pas nécessaire que les auteurs de schémas comprennent RegEx, mais les utilisateurs peuvent consulter les règles RegEx dans le référentiel GitHub format_options et demander les leurs en soulevant un problème ou en envoyant un e-mail à adc@uoguelph.ca.

Numérique

Comme pour DateTime, les utilisateurs peuvent sélectionner une règle de format dans le menu déroulant. En arrière-plan, les règles de format sont exprimées dans RegEx documenté dans GitHub.

Texte

Comme pour DateTime et Numeric, les règles de format sont sélectionnées dans une liste déroulante mais exprimées sous forme de RegEx et documentées dans le référentiel GitHub.

Tableaux

Pour les entrées de données qui sont des tableaux, les règles de format s'appliquent aux éléments individuels du tableau. Par exemple, la règle de format pour une seule entrée de texte est la même règle de format utilisée pour un tableau des mêmes entrées de texte.

Overlays Capture Architecture ne spécifie pas le séparateur de tableau requis pour les tableaux, car différents systèmes auront des représentations différentes pour les tableaux. Notre recommandation pour les données tabulaires Excel ou .csv est de séparer les données par un point-virgule (;) et d'inclure le texte entre guillemets (").

Règle de Format Personnalisée

Si vous ne sélectionnez pas une règle de format dans la liste déroulante, vous avez la possibilité d’écrire votre propre règle de format en expressions régulières, ou regex. Les regex sont des séquences de caractères qui définissent des motifs de recherche, principalement utilisées pour la correspondance et la manipulation de chaînes de texte. Elles permettent de trouver, extraire, remplacer ou valider du texte selon des critères complexes. Derrière les règles de format fournies se trouvent des regex, documentées dans le dépôt GitHub format_options.

Si vous choisissez de fournir votre propre regex, assurez-vous de vérifier son bon fonctionnement à l’aide de l’outil de vérification des données web du moteur sémantique afin de garantir que la regex fonctionne comme prévu.
