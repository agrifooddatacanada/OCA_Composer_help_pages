Création d'un schéma d'aide avec cette page


Si vous avez déjà un ensemble de données dans un fichier Excel ou .csv, vous pouvez faire glisser et déposer ce fichier sur la zone de la page principale. Vous ne téléchargerez aucun ensemble de données. Au lieu de cela, ce site Web lira le fichier et extraira les en-têtes (il suppose qu'il s'agit de la première ligne). Ces en-têtes sont automatiquement ajoutés en tant qu'attributs au schéma que vous créez.

En regardant cet exemple d'ensemble de données, nous pouvons voir que les en-têtes se trouvent sur la première ligne et que tous les noms d'attributs sont bien formés.

Voici une liste de bonnes pratiques recommandées à prendre en compte lors de la dénomination de vos attributs :

  • Définissez la longueur maximale sur 32 caractères
  • Commencez TOUJOURS les noms de variables par une lettre
  • Les nombres peuvent être utilisés n'importe où dans le nom de la variable APRÈS le premier caractère
  • Utilisez UNIQUEMENT des traits de soulignement « _ » dans un nom de variable
  • N'utilisez PAS d'espaces vides ou vides

Exemples de noms d'attributs :

  • Régime A → régime_a
  • Longueur des fibres en centimètres → fibre_cm
  • Emplacement de la ferme → emplacement
  • Prix payé pour la toison → prix
  • Poids mesuré pendant la 2e semaine d'essai → poids2

Lorsque vous utilisez le moteur sémantique, vous téléchargez le moteur pour qu'il s'exécute localement dans votre navigateur. Lorsque vous ajoutez un ensemble de données, il reste sur votre ordinateur et est utilisé sur votre machine par le code moteur que vous avez téléchargé. Vous ne téléchargez jamais votre ensemble de données sur nos serveurs lorsque vous utilisez le moteur sémantique et nous ne stockons jamais votre ensemble de données. À la fin, lorsque vous générez le schéma OCA, le schéma que vous avez écrit est téléchargé vers notre analyseur et converti en OCA que vous pouvez télécharger.

Vous pouvez également créer votre schéma manuellement, en écrivant chaque nom d'attribut via l'interface utilisateur.
