Superpositions - Séparateur de données


La superposition Séparateur de données vous permet de documenter la manière dont les valeurs de votre ensemble de données sont séparées et formatées. Cela est particulièrement important lors du partage de données dans des formats de fichiers texte tels que CSV ou TSV, ou lorsque vos données contiennent des nombres décimaux ou des attributs de type tableau. Chaque section peut être activée ou désactivée indépendamment à l'aide de sa case à cocher.

Séparateur décimal

Le séparateur décimal est le caractère utilisé pour distinguer la partie entière d'un nombre de sa partie fractionnaire. Deux conventions sont largement utilisées dans le monde :

  • Point ( . ) – utilisé dans les pays anglophones (ex. : 3.14).
  • Virgule ( , ) – utilisé dans de nombreux pays européens et latino-américains (ex. : 3,14).

Activez cette section et choisissez le séparateur qui correspond à la convention utilisée dans votre ensemble de données. Notez que le séparateur décimal et le délimiteur de champ ne peuvent pas être le même caractère ; l'interface empêchera automatiquement les sélections conflictuelles.

Délimiteur de fichier (CSV / TSV)

Un délimiteur de fichier définit la structure des champs et des lignes dans un fichier de données tabulaires en texte brut. Activez cette section pour documenter les paramètres de délimitation exacts utilisés par votre ensemble de données, afin que toute personne lisant ou traitant ce fichier puisse l'analyser correctement.

  • Délimiteur de champ – Le caractère qui sépare une colonne de la suivante. Les choix courants sont la virgule (,), la tabulation (\t), le point-virgule (;) et la barre verticale (|). La virgule est sélectionnée par défaut. Si vous avez défini la virgule comme séparateur décimal, l'option virgule est automatiquement retirée de cette liste pour éviter toute ambiguïté.
  • Caractère de citation – Un seul caractère utilisé pour encadrer les valeurs de champ contenant le délimiteur ou des sauts de ligne (défaut : "). Un seul caractère est accepté.
  • Caractère d'échappement – Un seul caractère utilisé pour échapper les caractères spéciaux à l'intérieur d'un champ entre guillemets (défaut : \). Un seul caractère est accepté.
  • Terminateur de ligne – La séquence de caractères utilisée pour marquer la fin de chaque ligne. Choisissez LF (\n) pour les fins de ligne Unix/macOS ou CRLF (\r\n) pour les fins de ligne Windows.
  • Ligne de début des données – Le numéro de ligne à partir duquel commencent les enregistrements de données (défaut : 1). Augmentez cette valeur si votre fichier comporte des lignes d'en-tête ou de métadonnées avant la première ligne de données.

Délimiteur de tableau

Lorsqu'un ou plusieurs attributs de votre schéma sont de type Tableau, chaque cellule de cette colonne contient plusieurs valeurs regroupées dans un seul champ. Le délimiteur de tableau est le caractère utilisé pour séparer ces valeurs individuelles à l'intérieur de la cellule.

Si votre schéma contient des attributs de type Tableau, cette section les listera chacun et vous permettra de choisir leur délimiteur de manière indépendante. Les options disponibles sont la virgule (,), la tabulation (\t), le point-virgule (;) et la barre verticale (|).

Si aucun de vos attributs n'est de type Tableau, cette section l'indiquera et les contrôles ne seront pas disponibles.
