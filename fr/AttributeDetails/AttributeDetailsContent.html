Détails de l'attribut

Sur la page des détails de l'attribut, vous pouvez modifier les propriétés de chaque attribut présent dans votre schéma.

Nom de l'attribut

Vous pouvez modifier chaque nom d'attribut et réorganiser les attributs.

Exigences : Les noms d'attributs DOIVENT contenir uniquement les caractères suivants : chiffres : 0-9, lettres : a-z et A-Z, soulignement : _, trait d'union : -, point :.

Recommandations> :

  • Définissez la longueur maximale sur 32 caractères
  • Commencez toujours les noms de variables par une lettre
  • Les nombres peuvent être utilisés n'importe où dans le nom de la variable APRÈS le premier caractère
  • N'utilisez pas d'espaces vides ou vides

Les noms d'attribut sont reconnus par différents programmes (lisibles par machine) et donc un bon nom d'attribut est celui qui ne causera pas de problèmes en aval.

Sensible

Les attributs sensibles vous permettent de spécifier un attribut sensible pour les données sensibles (par exemple, les noms de personnes, les pièces d'identité officielles, la date de naissance, etc.). Marquer une entrée avec une coche vous permettra de signaler tous les attributs dans lesquels des informations d'identification sur les entités peuvent être capturées.

  • Le fait de signaler un attribut agit comme une alerte aux utilisateurs de données pour qu'ils traitent les données comme étant à haut risque tout au long du cycle de vie des données et qu'ils les chiffrent ou les suppriment à tout moment.
  • Pour obtenir de l'aide sur l'évaluation des attributs qui peuvent être sensibles, vous pouvez lire les conseils dans le Rapport sur la taxonomie des identités aveuglantes de l'Initiative Kantara.

Unité

Les unités sont les endroits où vous pouvez décrire les unités de chaque attribut si nécessaire.

Type

  • Texte : texte, des entrées à une seule lettre aux champs de texte non structuré. Les nombres sont du texte si vous ne pouvez pas effectuer d'opérations mathématiques dessus, par exemple un numéro de téléphone est du texte.
  • Numérique : nombres. Si vous ne pouvez pas le mettre dans une « calculatrice » et effectuer directement des opérations mathématiques dessus, alors vous avez du texte. Par exemple, la plage « 1056-1067 » est du texte, la valeur a10 est du texte car elle contient une lettre dans le nom, etc.
  • Booléen : un type de données où les données n'ont que deux valeurs possibles : vrai ou faux
  • Fichier binaire : un type de données qui définit un signal de code binaire (un fichier binaire)
  • DateTime : un type de données qui définit les dates. Les formats courants incluent les dates (par exemple, AAAA-MM-JJ), les heures (par exemple, hh:mm:ss), les dates et heures concaténées (par exemple, AAAA-MM-JJThh:mm:ss.sss+zz:zz) et les durées (par exemple, PnYnMnD)
  • Tableau [type d'attribut] : un type de données qui définit une structure contenant plusieurs éléments de données ou éléments du même type de données

Liste

Pour mieux comprendre les listes (alias codes d'entrée), vous pouvez trouver un exemple concret dans notre article de blog sur les codes d'entrée.

Une liste est une fonctionnalité très utile qui permet à l'auteur du schéma de limiter les données qui peuvent être saisies pour un attribut spécifique. Par exemple, si votre variable comporte des catégories, vous souhaiterez spécifier vos catégories dans une liste. Sélectionnez une liste si vous souhaitez créer une liste d'entrées acceptables parmi lesquelles l'utilisateur pourra faire son choix.

Si l'utilisateur ne peut sélectionner qu'une seule entrée valide dans la liste (par exemple, une liste dans laquelle l'utilisateur sélectionnera l'emplacement de l'échantillon), votre type de données sera quelque chose d'approprié comme « Texte » ou « Numérique ». Si vous autorisez la sélection multiple dans la liste (par exemple, une liste dans laquelle l'utilisateur peut sélectionner tout ce qui s'applique), alors votre type de données est un tableau.
