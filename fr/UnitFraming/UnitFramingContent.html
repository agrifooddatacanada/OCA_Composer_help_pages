Overlays - Unit Framing


Pourquoi les unités normalisées sont importantes

Les unités peuvent être écrites de différentes manières. Par exemple, l'unité de concentration « micromolaire » peut apparaître sous la forme uM, μM, umol/L, μmol/l ou umol-1. Cette variation peut entraîner des confusions ou des erreurs d'interprétation des données.

Pour garantir la clarté, la cohérence et l'interopérabilité entre les ensembles de données et les systèmes, il est important d'utiliser des formats d'unités normalisés. Les normes permettent d'éliminer toute ambiguïté, notamment dans les environnements collaboratifs ou automatisés.

Le Code unifié des unités de mesure » ​​(UCUM) est un système largement adopté pour définir et coder les unités. UCUM fournit un format univoque et lisible par machine pour spécifier les unités, favorisant ainsi l'échange et l'interprétation précis des données.

Le moteur sémantique prend en charge les unités UCUM via un cadre d'unités superposé. Cela signifie que vous pouvez écrire les unités telles que vous les connaissez dans le cadre d'unités superposé, puis associer l'unité choisie (par exemple, acre) à la terminologie officielle UCUM (par exemple, [acr_us]) dans le cadre d'unités superposé.
