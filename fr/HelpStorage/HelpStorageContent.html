Sauvegarder, déposer et/ou publier votre schéma


Vous souhaiterez vous assurer que votre schéma est disponible pour référence et/ou réutilisation. Voici plusieurs options :

  • Vous pouvez enregistrer votre schéma avec vos données.
  • Vous pouvez déposer votre schéma avec un ensemble de données lorsque vous le placez dans un dépôt.
  • Vous pouvez publier votre schéma séparément, où il reçoit un identifiant que vous pouvez utiliser pour associer le schéma à vos ensembles de données.

Enregistrer votre schéma avec vos données

La première et la plus simple façon d'enregistrer votre schéma pour référence future est de le conserver avec l'ensemble de données auquel il est associé. Cela est particulièrement utile pour les schémas qui sont très spécifiques à votre ensemble de données et où vous n'avez pas créé le schéma en vue d'une réutilisation.

Nous vous suggérons d'enregistrer votre schéma à la fois comme modèle Excel et comme paquet OCA. Le paquet OCA est une version lisible par machine et une norme publiée. Il s'agit de la version la plus durable de votre schéma. Le modèle Excel est une version lisible par l'humain et utile à conserver avec vos données. À mesure que nous développons le moteur sémantique, nous abandonnerons le modèle Excel au profit d'autres outils pour générer des schémas OCA.

Déposer votre schéma séparément dans un dépôt

Les schémas peuvent être rédigés pour être utiles à plusieurs ensembles de données. Un schéma bien conçu peut être très utile pour une communauté de recherche car il peut faciliter le partage et la compréhension des données, et réduire le travail de documentation.

Vous pouvez déposer un schéma séparément dans le dépôt Borealis de l'Université de Guelph, ce qui attribuera un DOI citant à votre schéma. Vous pouvez partager et publier ce DOI et le référencer lorsque vous publiez vos données. D'autres chercheurs pourront référencer votre schéma pour leurs propres collectes et publications de données.

Obtenir un DOI via Borealis :

  1. Connectez-vous à Borealis, un dépôt Dataverse canadien, en utilisant votre nom d'utilisateur et mot de passe U de G.
  2. Demandez la permission de déposer dans les dépôts de données de recherche de l'U de G en écrivant à lib.research@uoguelph.ca.
  3. Une fois l'accès accordé, accédez au dépôt de données de recherche agro-environnementales - Agri-food Data Canada – collection Schemas.
  4. Cliquez sur le bouton « Ajouter des données » et sélectionnez « Nouvel ensemble de données » dans le menu déroulant.
  5. Dans la section Modèle d'ensemble de données, assurez-vous que « ADC Schema » est sélectionné dans le menu déroulant.
  6. Dans la section Métadonnées de citation, remplissez les informations requises (par exemple, titre, auteur, contact, description, mots-clés, nom du déposant).
  7. Préparez votre paquet OCA en « zippant » le fichier *.zip, c'est-à-dire en créant une archive zip de l'archive zip originale. Le paquet OCA doit maintenant être « double-zippé ».
    • Cette étape est nécessaire car Borealis décompresse automatiquement les archives zip, et le paquet OCA doit rester au format zip.
  8. Dans la section Fichiers, glissez-déposez votre paquet OCA au format double-zippé (*.zip) ainsi que le schéma au format readme archive (*.txt).
  9. Cliquez sur Enregistrer l'ensemble de données.
    • Après avoir enregistré l'ensemble de données, un DOI est créé et réservé pour l'enregistrement.
  10. Modifiez l'enregistrement brouillon (par exemple, informations descriptives (métadonnées) ou fichiers) si nécessaire.
  11. Cliquez sur le bouton « Publier l'ensemble de données » et sélectionnez « Soumettre pour révision ».
  12. Le personnel du dépôt (à la bibliothèque) examinera et publiera l'enregistrement.
    • Après publication, le DOI est résolu et devient un lien actif.

Pour référence : Recherche et bourse, 2020, « Comment déposer des données de recherche dans le dépôt de données de recherche agro-environnementales ou dans le dépôt de données de recherche de l'Université de Guelph », https://doi.org/10.5683/SP2/CPHFGA, Borealis

Gestion des versions

Pour des mises à jour mineures (comme la correction de fautes de frappe), nous recommandons d'utiliser la fonction de remplacement du dépôt.

  • Utilisez la fonction Remplacer. Cela remplace le fichier dans l'enregistrement et met la version à jour.
  • Les versions précédentes du fichier restent accessibles via l'onglet Versions de l'enregistrement.

Pour des mises à jour majeures, nous recommandons de créer un autre DOI et de déposer le schéma mis à jour. Ajoutez la citation (y compris le DOI) de la version originale du schéma dans le champ « Ensembles de données connexes » des métadonnées de l'enregistrement.
