Enregistrer, déposer et/ou publier votre schéma


Vous devez vous assurer que votre schéma est disponible pour consultation et/ou réutilisation. Voici plusieurs options :

  • Vous pouvez enregistrer votre schéma avec vos données
  • Vous pouvez déposer votre schéma avec un jeu de données lorsque vous le placez dans un dépôt
  • Vous pouvez publier votre schéma séparément, où un identifiant lui sera attribué pour l'associer à vos jeux de données

Enregistrer votre schéma avec vos données

La première façon, et la plus simple, d'enregistrer votre schéma pour consultation ultérieure est de l'enregistrer avec le jeu de données auquel il est associé. Ceci est particulièrement utile pour les schémas très spécifiques à votre ensemble de données, que vous n'avez pas créés en vue de les réutiliser.

Nous vous suggérons d'enregistrer votre schéma à la fois sous forme de modèle Excel et de fichier OCA. Le fichier OCA est une version lisible par machine et une norme publiée. Il devrait s'agir de la version la plus pérenne de votre schéma. Le modèle Excel est une version lisible par l'utilisateur et peut également être enregistré avec vos données. À mesure que nous développons le moteur sémantique, nous abandonnerons le modèle Excel et utiliserons d'autres outils pour générer des schémas OCA.

Déposer votre schéma séparément dans un dépôt

Les schémas peuvent être conçus pour être utilisés avec plusieurs ensembles de données. Un schéma bien écrit peut être très utile pour une communauté de recherche, car il facilite le partage et la compréhension des données, et simplifie la documentation.

Vous pouvez déposer un schéma séparément dans le dépôt Borealis de l'Université de Guelph, ce qui lui confèrera un DOI citable. Vous pouvez partager et publier ce DOI et le référencer lorsque vous publiez vos données. D'autres chercheurs peuvent référencer votre schéma pour leurs propres collectes et publications de données.

Obtenir un DOI via Borealis :

  1. Connectez-vous à Borealis, un dépôt Dataverse canadien, en utilisant votre nom d'utilisateur et votre mot de passe de l'Université de Guelph.
  2. Demandez l'autorisation de déposer des données dans les dépôts de données de recherche de l'Université de Guelph à lib.research@uoguelph.ca.
  3. Une fois l'accès accordé, accédez au dépôt de données de recherche agroenvironnementale - Données agroalimentaires Canada - Collection de schémas.
  4. Cliquez sur le bouton « Ajouter des données » et sélectionnez « Nouvel ensemble de données » dans le menu déroulant.
  5. Dans la section Modèle d'ensemble de données, assurez-vous que « Schéma ADC » est sélectionné dans le menu déroulant.
  6. Dans la section Métadonnées de citation, complétez les informations requises. (par exemple, titre, auteur, contact, description, mots-clés, nom du déposant).
  7. Préparez votre bundle de schéma OCA en « zippant » le fichier *.zip, c'est-à-dire en créant une archive zip de l'archive zip d'origine. Le fichier de schéma OCA doit maintenant être « double zippé ».
    • Cette étape est nécessaire car Borealis décompresse automatiquement les archives zippées et le fichier de schéma OCA doit être au format zippé.
  8. Dans la section Fichiers, glissez-déposez votre fichier de schéma OCA au format double zip (*.zip) et le schéma au format d'archive readme (*.txt).
  9. Cliquez sur « Enregistrer le jeu de données ».
    • Après avoir enregistré le jeu de données, un DOI est créé et réservé pour l'enregistrement.
  10. Modifiez le brouillon de l'enregistrement (par exemple, les informations descriptives (métadonnées) ou les fichiers) selon vos besoins.
  11. Cliquez sur le bouton « Publier le jeu de données » et sélectionnez « Soumettre pour révision ».
  12. Le personnel du dépôt (à la bibliothèque) examinera et publiera l'enregistrement.
    • Dès sa publication, le DOI est résolu et devient un lien fonctionnel.

Pour référence : Recherche et bourses, 2020, « Comment déposer des données de recherche dans le Dépôt de données de recherche agroenvironnementale ou dans le Dépôt de données de recherche de l’Université de Guelph », https://doi.org/10.5683/SP2/CPHFGA, Borealis

Versioning

Pour les mises à jour mineures (comme la correction d’erreurs typographiques), nous vous recommandons d’utiliser la fonction de remplacement du Dépôt.

  • Utilisez la fonction Remplacer. Ceci remplace le fichier dans l'enregistrement et crée une version de l'enregistrement.
  • Les versions précédentes du fichier restent accessibles via l'onglet « Versions » de l'enregistrement.

Pour les mises à jour majeures, nous vous recommandons de créer un autre DOI et d'y déposer le schéma mis à jour. Ajoutez la citation (y compris le DOI) à la version originale du schéma dans le champ « Ensembles de données associés » des métadonnées de l'enregistrement.
