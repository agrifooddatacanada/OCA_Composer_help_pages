Semantic Engine Changelog

Announcement

Coming early March, the Semantic Engine will move to OCA version 1.1 and introduce the OCA Package structure. For our users this means the Semantic Engine will produce and consume .jsons instead of .zips of schemas. All tooling continues to work the same and the Semantic Engine can continue to read and use .zip schemas. The file switch will help our Mac users who have some challenges using .zips and will also make it easier to deposit the schemas in Borealis (Dataverse) repositories.

With the introduction of OCA Package structure we can now preserve attribute order, meaning that users will see improvements where their column headers can be kept in the original order as recorded in the data. Finally, any users with tools that use the old .zip format will be able to export their files in legacy .zip format.

Changelog

Coming early March 2025 - Update to OCA Package

  • OCA schemas are now available as OCA Package in the format of a single .json file instead of the previous .zip file
  • OCA Package consists of two parts: the core OCA syntax as a Schema Bundle, and Extensions which contains custom overlays such as the ordering overlay
  • OCA Package now supports attribute and entry code ordering. Ordering is now user determined instead of being sorted alphabetically.
  • The Schema bundle (a component of package) follows the 1.1 release of the OCA Specification. Major changes of this include:
    • Type of all overlays increments to version 1.1
    • Array DataTypes syntax has changed, from "Array[Text]" to ["Text"] (etc.)
    • Label of the digest changes from "digest": to "d":
    • Attribute names are limited to the following characters: numbers: 0-9, letters: a-z and A-Z, underline: _ hyphen: - period:.
