Overlays - Data Separator


The Data Separator overlay lets you document how values in your dataset are separated and formatted. This is especially important when sharing data in flat-file formats such as CSV or TSV, or when your data contains decimal numbers or array-type attributes. Each section can be enabled or disabled independently using its checkbox.

Decimal Separator

The decimal separator is the character used to distinguish the integer part of a number from its fractional part. Two conventions are widely used around the world:

  • Dot ( . ) – used in English-speaking countries (e.g., 3.14).
  • Comma ( , ) – used in many European and Latin American countries (e.g., 3,14).

Enable this section and choose the separator that matches the convention used in your dataset. Note that the decimal separator and the field delimiter cannot be the same character; the interface will automatically prevent conflicting selections.

File Delimiter (CSV / TSV)

A file delimiter defines how individual fields and rows are structured in a plain-text tabular data file. Enable this section to document the exact delimiter settings used by your dataset so that anyone reading or processing it can parse it correctly.

  • Field delimiter – The character that separates one column from the next. Common choices are Comma (,), Tab (\t), Semicolon (;), and Pipe (|). The comma is selected by default. If you have set a comma as your decimal separator, the comma option is automatically removed from this list to prevent ambiguity.
  • Quote character – A single character used to wrap field values that contain the delimiter or line breaks (default: "). Only one character is accepted.
  • Escape character – A single character used to escape special characters within a quoted field (default: \). Only one character is accepted.
  • Line terminator – The character sequence used to mark the end of each row. Choose LF (\n) for Unix/macOS line endings or CRLF (\r\n) for Windows line endings.
  • Data start row – The row number where the actual data records begin (default: 1). Increase this value if your file has header or metadata rows before the first data row.

Array Delimiter

When one or more attributes in your schema are of an Array type, each cell in that column contains multiple values packed into a single field. The array delimiter is the character used to separate those individual values within the cell.

If your schema contains array-type attributes, this section will list each one and allow you to choose its delimiter independently. Available options are Comma (,), Tab (\t), Semicolon (;), and Pipe (|).

If none of your attributes are of an Array type, this section will indicate that and the controls will not be available.
